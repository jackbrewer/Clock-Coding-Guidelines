Overview

This document outlines core processes we have in place, to ensure our work is to the highest possible standard. It should also help create a default state for our code, to make it seamless when passing work between members of the production staff.

Browser Support

Unless client requirements stipulate otherwise, our full list of browser support is currently:

  • Chrome
  • Firefox
  • Opera
  • Safari
  • IE 7+

IE 6 should be considered, but does not need to function as fully as in the above — an acceptable level of usability is expected though. Everything we produce should be tested in all browsers available, on Windows, Mac as well as mobile browsers.

Text Editor Setup

Choice of application is up to you - we use Git and SVN for the majority of our projects so integration with these is useful. Most members of staff are currently using Sublime Text 2.

We use tabs for indentation, usually set to a double-space width. We also trim all trailing whitespace when saving (with the exception of Jade files). Please ensure these are both set correctly in your editor.

Jackbrewer on GitHub has made some great HTML and CSS bundles that speed up dev time in TextMate and Sublime. His Sublime settings are also available to aid with setup.
